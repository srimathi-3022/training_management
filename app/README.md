#  Training Management System API

A RESTful API built with FastAPI and MongoDB to manage trainees, courses, registrations, and attendance for a training institution.



##  Tech Stack

| Layer | Technology |
|---|---|
| Framework | FastAPI |
| Database | MongoDB |
| ODM Driver | PyMongo |
| Validation | Pydantic |
| Server | Uvicorn |
| Config | python-dotenv |


##  Project Structure

The project is organized into the following folders and files under the main `app/` directory:

- database.py — MongoDB connection and collection references
- models/ — Pydantic schemas for Trainee, Course, Registration, Attendance, and Score
- routes/ — API route handlers for each resource
- main.py — FastAPI application entry point and router registration



##  Setup & Installation

1. Clone the repository and navigate into the project folder.
2. Create and activate a Python virtual environment.
3. Install the required packages: `fastapi`, `uvicorn`, `pymongo`, `python-dotenv`, and `pydantic[email]`.
4. Create a `.env` file in the root directory and add your MongoDB connection URI as `MONGO_URI`.
5. Start the server using Uvicorn with the `--reload` flag for development.

Once running, the API is available at `http://127.0.0.1:8000` and the interactive Swagger docs are at `http://127.0.0.1:8000/docs`.


##  API Endpoints

###  Trainees

| Method | Endpoint | Description |
|---|---|---|
| POST | /trainees/ | Add a new trainee |
| GET | /trainees/{trainee_id} | Get trainee by ID |
| GET | /trainees/city/{city} | Get all trainees from a city |

Required fields when adding a trainee: trainee_name, trainee_age, trainee_email, trainee_phone, city, skills (list).



###  Courses

| Method | Endpoint | Description |
|---|---|---|
| POST | /courses/ | Add a new course |
| GET | /courses/{course_id} | Get course by ID |
Required fields when adding a course: course_name, trainer_name, course_duration, topics (list).


###  Registrations

| Method | Endpoint | Description |
|---|---|---|
| POST | /registrations/ | Register a trainee to a course |

Required fields: course_id, trainee_id. Both the trainee and course must exist before registering.


###  Attendance & Assessment

| Method | Endpoint | Description |
|---|---|---|
| POST | /attendance/mark | Mark attendance for a specific date |
| POST | /attendance/score | Add an assessment score |
| GET | /attendance/progress/{trainee_id}/{course_id} | Get full progress report |

Mark attendance fields: trainee_id, course_id, date (YYYY-MM-DD format), total_classes.

Add score fields: trainee_id, course_id, score (between 0 and 100). A score of 50 or above is a Pass; below 50 is a Fail.

Progress report returns the trainee's profile, course details, attendance percentage, and assessment result in one response.



## Recommended Testing Order

Follow this sequence to avoid errors caused by missing references:

1. Add a trainee — this generates a trainee_id
2. Add a course — this generates a course_id
3. Register the trainee to the course using both IDs
4. Mark attendance for one or more dates
5. Add an assessment score
6. Fetch the progress report to see the full summary



## Error Reference

| Status Code | Meaning | 
|---|---|
|400 | Bad request — duplicate email, already registered, or date already marked |
|404 | Resource not found — trainee, course, registration, or attendance record missing |
| 422 | Validation error — missing field, wrong data type, or score out of range |
|500 | Server error — usually a serialization issue with MongoDB |



##  Postman Tips

- Use Body -> raw -> JSON mode for all POST requests; the Content-Type header is set automatically.
- Field names must exactly match the Pydantic model — for example, use trainee_email not email.
- IDs are auto-generated by the server — do not include trainee_id or course_id in POST request bodies.
- Test the endpoints in the recommended order above to avoid 404 errors.
